# Глоссарий терминов

## Роутинг (hash-based navigation)

**Простыми словами:** Как сайт понимает, какую страницу показать.

**Пример:**
- Пользователь кликает на ссылку "Услуги"
- В адресной строке появляется `#/services`
- Сайт показывает страницу с услугами

**Зачем hash (`#`):**
- Статический сайт (без сервера)
- Не требует перезагрузки страницы
- Работает на GitHub Pages

**Как это выглядит:**
```javascript
// Когда пользователь кликает на ссылку
window.location.hash = '#/services';

// Сайт слушает изменения hash и показывает нужную страницу
window.addEventListener('hashchange', () => {
  const route = window.location.hash.slice(1); // убираем #
  showPage(route); // показываем страницу
});
```

## Компоненты (без стилей, только логика)

**Простыми словами:** Переиспользуемые кусочки интерфейса, которые делают что-то конкретное.

**Пример:**
- Компонент "Карточка услуги" — показывает название, описание, кнопку
- Компонент "Навигация" — показывает меню с ссылками
- Компонент "Поиск" — ищет по базе знаний

**"Без стилей, только логика" означает:**
- Компонент знает ЧТО делать (показать данные, обработать клик)
- Компонент НЕ знает КАК выглядеть (цвета, шрифты, отступы)

**Зачем так:**
- Можно менять дизайн отдельно от логики
- Легче тестировать
- Легче переиспользовать

**Пример компонента:**
```javascript
// Компонент "Карточка услуги" (только логика)
function ServiceCard(service) {
  return {
    title: service.title,
    description: service.description,
    onAddToCart: () => {
      // Логика добавления в корзину
      addToArtifacts(service);
    }
  };
}

// Стили применяются отдельно в CSS
```

## Рабочий интерфейс с разделами и подразделами

**Это то, что ты хочешь:** Структура сайта, которую легко менять.

**Что это значит:**
- Разделы: Главная, Услуги, Портфолио, База знаний
- Подразделы: внутри каждого раздела могут быть свои страницы
- Легко менять: структура берётся из `routes.json`, не нужно править код

**Пример структуры:**
```
/ (Главная)
├── /services (Услуги)
│   ├── /services/video (Видеопродакшн)
│   └── /services/design (Дизайн)
├── /portfolio (Портфолио)
│   └── /portfolio/[slug] (Детальная страница кейса)
└── /kb (База знаний)
    └── /kb/[slug] (Детальная страница термина)
```

**Как это реализовать:**
1. Структура берётся из `routes.json` (из core)
2. Компонент "Навигация" читает структуру и показывает меню
3. Компонент "Роутер" показывает нужную страницу по hash
4. Чтобы изменить структуру — меняешь `routes.json` в core, переимпортируешь

**Это и есть "Компоненты (без стилей, только логика)":**
- Компонент "Навигация" — знает, как построить меню из routes.json
- Компонент "Роутер" — знает, какую страницу показать по hash
- Компонент "Страница" — знает, как отобразить контент

## Связь с ядром (core)

**Ядро (vovaipetrova-core):**
- Хранит контент (Markdown файлы)
- Генерирует данные (routes.json, kb_glossary_lite.jsonl)
- Экспортирует в site через `npm run import`

**Сайт (vovaipetrova-site):**
- Получает данные из core
- Показывает интерфейс пользователю
- Не хранит контент сам

**Как это работает:**
1. В core меняешь структуру (routes.json)
2. В site запускаешь `npm run import`
3. Сайт получает новую структуру
4. Интерфейс автоматически показывает новую структуру
